## Lista de Verificação



<center>

**Tabela 1:** Lista de verificação para Requisitos Elicitados.

Aqui está a tabela com a coluna de observação preenchida para cada pergunta, incluindo a contextualização do motivo da pergunta:

| ID  | Pergunta     | Observação    | Rastreabilidade |
|-----|---------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------|
| 1  |  Os requisitos elicitados foram identificados com base nas necessidades e objetivos dos usuários, conforme recomendado na literatura? | Identificar requisitos com base nas necessidades dos usuários garante que o sistema atenda às expectativas e resolva problemas reais. | SILVA, J.; OLIVEIRA, M. Interação humano-computador e experiência do usuário. Editora Tech, 2020. Pág 166. |
| 2  |  Os requisitos elicitados foram definidos com rigor e detalhes, mantendo o foco nas necessidades dos usuários, como recomendado na literatura? | Requisitos bem definidos e detalhados evitam ambiguidades e garantem que o desenvolvimento seja alinhado às necessidades dos usuários. | SILVA, J.; OLIVEIRA, M. Interação humano-computador e experiência do usuário. Editora Tech, 2020. Pág 176. |
| 3  |  Os requisitos elicitados foram classificados de acordo com sua natureza (funcionais, não funcionais, etc.) para facilitar a compreensão e implementação, conforme sugerido na literatura? | Classificar os requisitos facilita a sua gestão e implementação, garantindo que todos os aspectos do sistema sejam cobertos. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 140. |
| 4  |  Os requisitos elicitados foram documentados de forma a permitir sua rastreabilidade ao longo do ciclo de vida do sistema, facilitando futuras atualizações e manutenções, conforme descrito na literatura? | Rastreabilidade dos requisitos permite acompanhar mudanças e garantir que todos os requisitos sejam atendidos durante o desenvolvimento. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 266. |
| 5  |  Os requisitos elicitados foram classificados em implementados ou não? | Classificar os requisitos em implementados e não implementados ajuda a monitorar o progresso do projeto e a gerenciar prioridades. | SILVA, J.; OLIVEIRA, M. Interação humano-computador e experiência do usuário. Editora Tech, 2020. |
| 6  | Os requisitos são testáveis e mensuráveis para garantir que possam ser verificados durante a implementação?  | Requisitos testáveis e mensuráveis garantem que o sistema possa ser validado contra as especificações, assegurando a qualidade do produto final. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 266. |
| 7  |  Todos os requisitos foram priorizados de acordo com sua importância para o sucesso do projeto? | Priorizar requisitos ajuda a focar no desenvolvimento das funcionalidades mais críticas para o sucesso do projeto. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 22. |
| 8  |  Os requisitos são consistentes e não entram em conflito uns com os outros? | Requisitos consistentes evitam contradições que poderiam comprometer o desenvolvimento e a funcionalidade do sistema. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 140. |
| 9  |  Os requisitos consideram os requisitos não funcionais, como desempenho, segurança e usabilidade? | Considerar requisitos não funcionais garante que o sistema não apenas funcione corretamente, mas também atenda a padrões de qualidade, segurança e usabilidade. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 140. |
| 10  |  Os requisitos foram documentados de forma abrangente e estão prontos para serem utilizados na fase de desenvolvimento do software? | Documentar requisitos de forma abrangente assegura que todos os detalhes necessários estão disponíveis para a equipe de desenvolvimento, evitando retrabalho e mal-entendidos. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 266. |
| 11 | Os requisitos abordam requisitos de conformidade, como regulamentações legais ou padrões da indústria que o sistema deve atender? | Considerar requisitos de conformidade garante que o sistema estará em conformidade com as leis e regulamentos aplicáveis, evitando problemas legais. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 49. |
| 12 | Os requisitos incluem requisitos de escalabilidade para garantir que o sistema possa lidar com um aumento no número de usuários? | Incluir requisitos de escalabilidade assegura que o sistema pode crescer e se adaptar ao aumento de demanda sem perda de performance. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 49. |
| 13 | Os requisitos consideram os requisitos de segurança, como autenticação, autorização e criptografia de dados? | Requisitos de segurança são essenciais para proteger os dados dos usuários e garantir a integridade do sistema. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 140. |
| 14 | Os requisitos abordam os requisitos de desempenho, como tempos de resposta esperados e capacidade de processamento? | Abordar requisitos de desempenho garante que o sistema atenderá às expectativas de rapidez e eficiência dos usuários. | WIEGERS, Karl; BEATTY, Joy. Software Requirements. 3. ed. Redmond: Microsoft Press, 2013. Pág 266. |
