## Introdução
Este documento apresenta uma lista de verificação para a avaliação do tópico da técnica de elicitação de requisitos de observação do projeto dos Correios. Dessa maneira, através da lista de verificação, será buscado identificar de forma objetiva e detalhada os requisitos relacionados à técnica de observação ao garantir que eles estejam alinhados com as necessidades e com as expectativas dos usuários finais.

## Metodologia 
Para conduzir a avaliação, inicialmente, será elaborado uma lista de verificações com base em materiais didáticos sobre a técnica de elicitação de requisitos de observação. Em seguida, um integrante do grupo irá revisar essa especificação com base nesta lista.

## Lista de Verificação

A tabela 1 apresenta a lista de verificação da técnica de Observação com base em pontos citados em livros e artigos.

<center>

**Tabela 1:** Lista de verificação para a técnica de observação.

| ID  | Pergunta  | Observação  | Rastreabilidade | Captura de tela |
|-----|-----------|-------------|-----------------|-----------------|
| 1  | O documento descreve a metodologia de observação utilizada (passiva, participativa, etc.)? | Deve especificar o tipo de observação (ex.: passiva, onde o observador não interage, ou participativa, onde o observador interage com os participantes). |  |  |
| 2  | O documento explica os critérios para selecionar os cenários e tarefas observados? | Necessário definir como foram escolhidos os cenários e as tarefas a serem observados para garantir que sejam representativos. |   |  |
| 3  |O documento detalha como foram registradas as observações (anotações, gravações, etc.)?| Deve descrever os métodos de registro utilizados, como anotações manuais, gravações de áudio/vídeo, etc. |  |  |
| 4  |O documento inclui uma descrição dos comportamentos e interações observados?| É importante documentar detalhadamente os comportamentos e as interações dos usuários durante a observação. |  |  |
| 5  |O documento identifica discrepâncias entre o comportamento esperado e o comportamento observado?| Deve apontar diferenças entre como os usuários deveriam se comportar (segundo os requisitos) e como realmente se comportam. |  |  |
| 6  | O documento apresenta insights ou descobertas relevantes obtidos durante a observação?|Necessário destacar descobertas importantes que possam influenciar o desenvolvimento dos requisitos.|  |  |
| 7  | O documento discute como as descobertas da observação foram traduzidas em requisitos de software? | Deve explicar como as informações coletadas foram analisadas e convertidas em requisitos concretos. |  |  |
| 8  |O documento menciona se houve a validação das observações com os participantes? |Importante confirmar se as observações foram validadas pelos próprios usuários para assegurar a precisão. |  |  |
| 9  |O documento contém exemplos específicos de como as observações impactaram o design do sistema? |Necessário incluir exemplos concretos de mudanças no design do sistema baseadas nas observações.|  |  |
| 10  | O documento apresenta uma análise de pontos críticos ou gargalos identificados durante a observação?| Deve identificar e analisar pontos onde ocorrem problemas ou atrasos significativos no processo observado.|  |  |
| 11  |O documento descreve as condições ambientais e contextuais durante a observação?|Importante detalhar as condições ambientais (ex.: local, horário) que podem ter influenciado as observações.|  |  |
| 12  |O documento inclui uma avaliação da eficácia da técnica de observação utilizada?| Deve avaliar se a técnica de observação aplicada foi eficaz em obter as informações necessárias.|  |  |
| 13  |O documento menciona limitações da observação e como foram mitigadas?|Necessário discutir limitações encontradas durante a observação e as estratégias adotadas para mitigá-las.|  |  |
| 14  |O documento descreve como as observações foram revisadas e integradas com outros métodos de elicitação?|Importante explicar como as informações obtidas por observação foram combinadas com dados de outras técnicas (ex.: entrevistas, questionários).|  |  |
| 15  |O documento apresenta um resumo das principais lições aprendidas durante o processo de observação?|Deve incluir um resumo das lições aprendidas para melhorar futuras observações e a elicitação de requisitos. |  |  |
